webhouse_project_x — Database Relationships

57 tables · Source: db/schema.sql

Central hub: wh_tenants — almost every business table links here via tenant_id.
Global tables (no tenant): wh_admin_users, modules, wh_subscription_plans
How to read: table.column → parent_table means a foreign key. Rows marked UNIQUE are one-to-one or unique per tenant. Soft delete: every table has deleted_at (NULL = not deleted). Auto-purge after 7 days is planned in app logic, not in DB yet.
Foreign keys: all use ON DELETE CASCADE and ON UPDATE CASCADE.
The DB does not enforce that tenant_id matches other FKs on the same row — the app must validate that.

High-level flow (text)

FlowPath
Subscribe tenant wh_subscription_plans → wh_subscription_module → copy to wh_tenant_modules + wh_tenant_subscriptions
Onboard tenant user wh_tenants → roles → users → permissions
Fulfill order crm_customers → orders → order_items → inventory_products
E-commerce sync ecom_store_connections → ecom_external_orders → orders
Courier pickup logistics_courier_partners → logistics_pickup_requests → logistics_pickup_orders → orders
POS sale pos_outlets → pos_terminals → pos_sales → pos_sale_items
Stock transfer inventory_stock_transfers → app writes 2× inventory_stock_movements → updates inventory_stock_levels
